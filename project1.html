    MAKING OF THE WEBSITE

    I convinced myself that I would follow, in detail, the project description by reading it for the 2nd time and began working on my masterpiece of a webpage. I´ve always been keen on nice interfaces, so by the time I was deciding whether I should choose one of the given templates provided by @ajlkn or start coding from scratch, I was positive that either way, I´d emphasize on making a nice looking site. Soon later I stumbled upon a project done by a previous student of this course, Birgir Steinn Hermannsson. Haflidi Asgeirsson, the supervisory teacher of the course, had posted two exemplary projects previously done, one of them being Birgir´s. I took a look at his contribution for this specific project and decided to use it as an inspiration. Not only did I find his work practical, but my requirements of how I wanted my website to look like fit perfectly with the interface he ended up with making. Credit to him.

    The core of my HTML script and .css file are very similar to the ones Birgir uploaded to GitHub. I began this process by writing similar codes using brackets with the intention of understanding every aspect of them in order to be able to create a website that would furthermore fit into my field of ongoing build of requirements. I am, to some extent, satisfied with this website.

    How I uploaded my website

    I uploaded the website through the GitHub desktop app.

    These are the steps I followed in order to do so:

    1. Create repository on the Github on the website (make sure that it is public).

    2. Clone the repository on Github Desktop.

    3. Transfer Brackets files into the repository on your desktop that mirrors the one you created on the website.

    4. Commit to master.

    5. Push Origin.

    6. On the Github website go to your repository.

    7. Go to settings. Scroll down till you see Github Pages. Select Brancch as master and file as /(root).

    8. Press save.

    9. You should now have a link to your working website.
